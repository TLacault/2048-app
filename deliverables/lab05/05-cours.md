# Lab 5 - Cloud

### Modèles de service cloud

- **IaaS** : Infrastructure as a Service (ex: AWS EC2) — accès à des machines virtuelles, gestion de l'OS et du runtime par l'utilisateur

- **PaaS** : Platform as a Service (ex: Vercel, Heroku) — abstraction de l'infrastructure, focus sur le code, déploiement simplifié

- **SaaS** : Software as a Service (ex: Google Docs) — application complète accessible via un navigateur, sans gestion technique

- **FaaS** : Function as a Service (ex: AWS Lambda) — exécution de fonctions en réponse à des événements, facturation à l'exécution

- **CaaS** : Container as a Service (ex: AWS ECS) — gestion de conteneurs, orchestration simplifiée, plus de contrôle que PaaS

### Modèles de déploiement cloud

- **Serverless** : modèle d'exécution où le fournisseur gère l'infrastructure, permettant de se concentrer sur le code sans se soucier de la gestion des serveurs

- **Cloud public** : services accessibles à tous via Internet (ex: AWS, Azure, GCP)

- **Cloud privé** : infrastructure dédiée à une organisation, souvent hébergée sur site ou dans un datacenter privé

- **Cloud hybride** : combinaison de cloud public et privé, permettant de répartir les workloads selon les besoins

- **Multi-cloud** : utilisation de plusieurs fournisseurs de cloud pour éviter la dépendance à un seul fournisseur et optimiser les coûts et les performances

- **Edge computing** : traitement des données à proximité de leur source pour réduire la latence et améliorer les performances, souvent utilisé pour les applications en temps réel

### Avantages
- Scalabilité automatique
- Maintenance réduite
- Déploiement rapide
- Infrastructure évolutive & scriptable
- Services clé en main (bases de données, authentification, etc.)

### Inconvénients
- Moins de contrôle sur l'infrastructure
- Coût potentiellement plus élevé à grande échelle
- Vendor lock-in (dépendance au fournisseur)
